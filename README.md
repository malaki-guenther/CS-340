# CS-340

I created maintainable, readable programs by using object-oriented design and separation of concerns. I built the AnimalShelter class in CRUD_Python_Module.py to handle all MongoDB CRUD operations, including automatic record numbering. This allowed the Dash dashboard in Project Two to stay clean, calling simple methods like db.read(query) instead of raw database code. The modular approach improved readability, simplified debugging, and makes the module reusable for future animal management systems.

As a computer scientist, I approach problems by developing and testing the data layer first, then building the interface. For Grazioso Salvare, I implemented rescue type filtering (Water, Mountain, Disaster), an interactive data table, pie chart, and map in the dashboard. This project required integrating MongoDB, Python, and Dash, a step up from previous isolated assignments. I will continue using layered development and client specific queries in future work.

Computer scientists build tools that turn data into actionable insights. This dashboard helps Grazioso Salvare staff quickly filter and visualize animals for rescue, improving efficiency and supporting faster adoption decisions.
